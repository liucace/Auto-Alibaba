# 1688 多品牌风扇 GEO 标题、SKU 与详情页修订设计

## 1. 目标

将 `DP201AT-2122HBL.GN` 的标题、1688 产品规格、SKU 和详情页改为可复用的多品牌风扇发布方案。后续同一流程必须支持 SUNON（建准）、Delta（台达）、Sanyo Denki（山洋）、Multifan、ebm-papst 及其他风扇品牌，而不是为 SUNON 或 ebm-papst 写单独分支。

操作者每个型号只提供：

1. 当前型号规格书 PDF；
2. 当前型号真实产品照片；
3. Excel 中当前完整型号对应的价格与库存。

品牌、完整型号、标题、属性、规格、详情内容、图片角色和尺寸图均由自动化从以上资料中提取。规格书和图片没有提供的普通技术值不填写、不补默认值、不从相近型号推测。

本次只修订准备、生成、填表和校验逻辑。上传仍通过专用 Chrome CDP 9223 串行运行，并严格停在“保存草稿”按钮之前；不得点击保存、发布或等价按钮。

## 2. 已确认的设计选择

### 2.1 详情方向

采用“证据优先、答案优先”的 GEO 详情结构。页面先直接回答产品是什么、完整型号是什么、关键参数是什么，再展示工作点、实物、尺寸、材料、采购核对和常见问题。

不采用以下方案：

- 单纯堆叠产品图片；
- 为了丰富页面而生成无来源的应用场景、认证、寿命、质保或营销数据；
- 为每个品牌维护一套独立详情模板；
- 把 50Hz/60Hz 工作点拆成两个 SKU。

### 2.2 跨品牌公司介绍尾图

## 3. 当前型号的确定输出

### 3.1 标题

`DP201AT-2122HBL.GN` 使用以下标题：

> SUNON DP201AT-2122HBL.GN 220-240V 120mm滚珠轴承交流轴流风扇

标题遵守当前 1688 标题输入框实测的 60 加权字符上限：ASCII 字符计 1，中文字符计 2。该标题普通字符数为 49、1688 加权计数为 59。原含“建准”的 51 字符候选标题被平台计为 63 并截断产品名称，因此不能使用。标题结构为：

`证据品牌标识 + 完整型号 + 额定电压 + 名义尺寸 + 轴承/关键结构 + 通用产品名称`

未来型号按同一语义结构生成，但只选取规格书或清晰铭牌中存在、对采购检索有帮助且能在 60 加权字符内表达的字段。不得写“最强”“原装正品”等无证据词，也不得因字符不够截断完整型号或通用产品名称。

标题与 1688 页面中的品牌、SKU、产品规格和详情首屏必须一致。

### 3.2 单一 SKU

当前商品只有一个 SKU：

| 1688 列 | 值 |
| --- | --- |
| 规格型号 | `DP201AT-2122HBL.GN` |
| 可售数量 | Excel 中该完整型号的库存，当前为 `10` |
| 单品货号 | `DP201AT-2122HBL.GN` |
| 是否上架 | 上架 |

买家选择 SKU 时看到的是完整规格型号 `DP201AT-2122HBL.GN`。`18/16.5W`、`2150/2500r/min`、`66/80CFM` 等斜杠值表示同一型号在 50Hz/60Hz 下的两个工作点，不是两个销售变体，不能拆成两个 SKU，也不能写进 SKU 名称制造伪变体。

未来每个上传任务默认仍是一个完整型号对应一个 SKU。只有外部输入和当前商品资料明确描述多个真实可售变体时，才允许另行设计多 SKU；本次范围不包含多 SKU。

### 3.3 1688 产品规格

当前页面七个规格字段填写如下：

| 字段 | 值 | 来源/说明 |
| --- | --- | --- |
| 规格型号 | `DP201AT-2122HBL.GN` | PDF 同页 `MODEL DP201AT` 与 `P/N 2122HBL.GN` 组合确认 |
| 电机功率 | `18/16.5 W` | PDF 50/60Hz 工作点 |
| 风叶直径 | `0.119 m` | PDF 图纸外框宽度 `119±0.5mm` 的平台字段映射 |
| 转速 | `2150/2500 r/min` | PDF 50/60Hz 工作点 |
| 风量 | `112.1/135.9 m³/h` | PDF 原值 `66/80 CFM`，按 `1 CFM ≈ 1.699 m³/h` 换算 |
| 电流 | `0.09/0.09 A` | PDF 50/60Hz 额定电流 |
| 重量 | `0.295 kg` | PDF 原值 `295g` |

`风叶直径` 是 1688 当前页面字段名称，PDF 实际证明的是外框宽度。详情正文必须准确写成“外框宽度 119±0.5mm”，不能把它描述为实测叶轮直径。若未来某型号规格书既没有叶轮直径，也没有可批准的平台尺寸映射，则该字段留空；不得从产品照片估算。

单位换算必须保留：原始值、换算值、换算公式或固定系数。详情页优先同时展示原始单位和平台单位，避免换算值被误解为规格书原文。

## 4. 证据与通用数据模型

### 4.1 证据来源优先级

1. 规格书中的制造商、品牌标识、型号表和技术参数；
2. 清晰铭牌或产品标签；
3. 当前型号实物照片中的可直接确认信息；
4. Excel 仅提供完整型号、价格和库存。

不得用目录名、历史商品、默认品牌、固定详情尾图或相近型号补齐当前产品信息。

### 4.2 工作点数据

50Hz 和 60Hz 参数应保存为结构化工作点，而不是无法解释的普通字符串。渲染 1688 单格字段时可以按平台允许的斜杠格式展示；渲染详情页时使用对照表明确频率与对应值。

当前型号工作点：

| 参数 | 50Hz | 60Hz |
| --- | --- | --- |
| 额定电压 | 220-240VAC | 220-240VAC |
| 转速 | 2150RPM | 2500RPM |
| 风量 | 66CFM / 112.1m³/h | 80CFM / 135.9m³/h |
| 最大静压 | 0.14inH₂O | 0.17inH₂O |
| 额定电流 | 0.09A | 0.09A |
| 功率 | 18W | 16.5W |
| 噪声 | 44dB(A) | 48dB(A) |

### 4.3 自适应模块

所有品牌共用一个详情渲染器。每个模块只依赖一个明确的证据结构，并在证据不足时整体省略：

- 电气模块：电压、频率、相数、电流、功率；
- 性能模块：转速、风量、静压、噪声；
- 结构模块：外框、厚度、孔距、孔径、叶轮、框体、轴承、引线；
- 环境模块：工作温度、存储温度、防护或绝缘信息；
- 安装模块：安装方向、气流方向、旋转方向；
- 安全/认证模块：只有规格书明确列出时才出现；
- FAQ：答案只能重组当前证据，不写通用但未经证明的承诺。

SUNON、Delta、Sanyo Denki、Multifan 和 ebm-papst 不建立品牌专用硬编码分支。不同产品类型通过证据决定模块组合，例如轴流风扇、离心风机、AC 风扇和 DC 风扇可以使用不同的已存在字段，但仍走同一渲染流程。

## 5. 详情页结构

详情内容宽度按当前 1688 TinyMCE 编辑区实测的约 790px 设计，使用内联样式和兼容的 HTML 表格，不依赖外部 CSS 或 JavaScript。

当前型号模块顺序固定为：

1. **答案优先首屏**：品牌、完整型号、产品定义和一句采购结论；
2. **快速确认**：电压、频率、尺寸、轴承、重量等最常查字段；
3. **型号与核心参数确认**：完整型号及重要规格；
4. **50/60Hz 工作点对照**：避免斜杠值含义不清；
5. **实物与铭牌核对**：四张当前型号真实照片，带语义化说明；
6. **尺寸、安装与气流方向**：PDF 尺寸图及图纸数据；
7. **材料、电气与环境信息**：只展示 PDF 已给出的值；
8. **采购前核对清单**：型号、电压/频率、尺寸/孔位、工作点；
9. **FAQ**：用直接问答解释型号、频率、风量换算和安装边界；
10. **公司介绍与服务能力**：固定六张公司介绍图，永远位于最末尾。

### 5.2 图片规则

当前型号主体使用：

- 一张从当前 PDF 指定页面、指定裁剪区域生成的尺寸图；
- 四张当前完整型号真实产品照片；
- 六张固定公司介绍图。

因此当前详情预计包含 11 个 `<img>`。实现不得继续把 `5` 或 `11` 写死为全局常量；应根据当前渲染计划计算期望图片数，并校验每个产品/尺寸图 URL 唯一、固定尾图齐全且顺序正确。

产品图 `alt` 使用“品牌 + 完整型号 + 图片角色”。公司图 `alt` 只写“公司介绍与服务能力 + 序号”，不写当前商品品牌和型号，以免被解释为产品证据。

不生成产品照片，不用其他型号照片补位，不覆盖原始照片。主图仍由本地准备器做确定性的等比缩放和补白。

## 6. 标题生成规则

标题生成器使用受限槽位，而不是自由营销文案：

1. 品牌规范名，必要时附规格书能确认的常用中文名；
2. 完整业务型号，绝不截断；
3. 额定电压或电压制式；
4. 采购最常用的名义尺寸；
5. 一个高价值结构特征，如滚珠轴承；
6. 准确的通用产品名称。

当 1688 加权总长度超过 60 时，按以下顺序删减：低价值修饰语、次要结构特征、尺寸的冗余表达、品牌中文别名。完整型号和通用产品名称不得删除。

生成后必须验证：

- 长度不超过 60；
- 完整型号出现且只出现一次；
- 品牌与证据一致；
- 不包含固定公司尾图中的其他品牌；
- 标题中的每个技术值可追溯到当前型号证据；
- 与 1688 属性、产品规格、SKU 和详情首屏一致。

## 7. 1688 页面填充与买家展示

### 7.1 页面一致性

填表器按当前真实 1688 模块区分三类内容：

- 商品属性/产品规格：技术字段；
- SKU 表：销售变体、库存、单品货号和上架状态；
- 商品详情：解释参数语义、证据、图片和采购核对。

不能把技术规格混入 SKU 名称，也不能因为 SKU 只有一个而复制多个假规格行。

### 7.2 详情同步

TinyMCE 写入后仍需同步编辑器内容和 React 表单模型。质量检查读取的 `description.detailList[0].content` 必须是完整 HTML，不得是 `null` 或字符串 `"null"`。

校验至少包括：

- 标题、品牌、完整型号和 SKU 一致；
- 七个当前产品规格值与计划一致；
- 当前型号产品图和尺寸图均存在；
- 六张公司介绍图均存在且位于最后；
- 公司介绍区域之前没有错误的 ebmpapst 产品归属；
- 页面阻断错误为零；
- 保存草稿按钮仍存在且未点击。

## 8. 品牌相册与满册处理

每个产品品牌使用独立连续编号相册：`品牌(NN)`，例如 `SUNON(01)`、`Delta(01)` 和 `Sanyo Denki(01)`。

上传当前产品图片时：

1. 只匹配当前品牌的严格相册名；
2. 优先选择当前品牌编号最大的相册；
3. 没有相册时创建 `(01)`；
4. 只有在 1688 明确返回容量不足/满册状态时，创建下一个连续编号；
5. 新相册创建成功后仅重试当前批次一次；
6. 新相册仍不可用、名称未保存或图片数量不正确时返回 `BLOCKED`；
7. 不删除、重命名或整理已有相册；
8. 不把当前品牌图片放入其他品牌相册。

固定六张公司介绍图使用现有托管 URL，不参与当前品牌相册选择和满册计数。

## 9. 错误处理

以下情况立即阻止准备或上传：

- 无法从当前 PDF/铭牌确认品牌或完整型号；
- PDF、照片、Excel 的完整型号冲突；
- 真实产品照片少于要求或无法确认是当前型号；
- 标题超过 60 加权字符或丢失完整型号；
- 1688 当前模块结构与计划字段不一致；
- 必填字段没有证据，或单位换算无法追溯；
- SKU 行数、型号、库存或货号与计划不一致；
- 详情图片缺失、重复、顺序错误或 TinyMCE/React 同步失败；
- 相册创建/满册重试失败；
- 页面出现新的阻断错误；
- 安全边界无法确认。

普通可选技术字段缺失不构成错误：该字段在属性、规格表、详情模块和 FAQ 中同时省略。

## 10. 测试与验收

实现采用测试驱动方式，至少覆盖：

- 当前 SUNON 标题准确且 1688 加权计数不超过 60；
- Delta、Sanyo Denki、Multifan 和 ebm-papst 证据不会走 SUNON 专用分支；
- 标题只使用当前证据，字符超限时按固定优先级删减；
- 单 SKU 使用完整型号，50/60Hz 斜杠值不产生第二 SKU；
- 七个当前规格字段及单位换算准确；
- 未提供的可选值不生成表单条目和详情空行；
- 工作点正确对齐 50Hz/60Hz；
- GEO 详情模块按证据出现或省略；
- 当前型号详情包含一张尺寸图、四张实物图和六张固定公司图；
- 固定公司图始终在最末尾且顺序不变；
- 非 ebmpapst 商品的产品主体不因固定尾图被改写为 ebmpapst；
- 图片期望数量由渲染计划计算，不依赖硬编码 5；
- 各品牌相册隔离、满册创建下一编号、只重试一次；
- TinyMCE 内容、React 模型和质量检查请求一致；
- 自动化停在“保存草稿”之前，不调用保存或发布动作。

离线验证运行完整单元/集成测试、静态检查和类型检查。真实页面验收只允许通过专用上传入口重新准备并填写 `DP201AT-2122HBL.GN`，确认 `READY_TO_SAVE`、阻断错误为零、所有字段和详情与本设计一致，然后停在保存草稿之前。

## 11. 设计依据

- 当前 1688 发布页实测：标题输入框 `maxlength=60`，同时使用 ASCII=1、中文=2 的加权计数；产品规格为七个独立字段；SKU 表列为“规格型号、可售数量、单品货号、是否上架”；TinyMCE 可用内容宽度约 790px。
- 1688 属性填写的通用要求强调属性真实准确、必填项完整，以及标题、图片、属性和详情一致：[1688 属性填写规则转载](https://www.weibo.com/ttarticle/p/show?id=2309404976003205431410)。
- 1688 标题长度与常见结构的辅助资料：[卖家网](https://m.maijiaw.com/article/603793)、[10100](https://www.10100.com/article/49844676)。页面实测结果优先于辅助资料。
- 产品实体、品牌、SKU、描述和附加属性的结构化表达参考：[Google Product 结构化数据](https://developers.google.com/search/docs/appearance/structured-data/product)、[Google Merchant Listing](https://developers.google.com/search/docs/appearance/structured-data/merchant-listing)、[Schema.org ProductModel](https://schema.org/ProductModel) 和 [additionalProperty](https://schema.org/additionalProperty)。
- GEO 内容采用明确实体、约束、意图和直接答案的设计思路，并以当前商品证据为边界：[E-GEO 论文](https://arxiv.org/abs/2511.20867)。
