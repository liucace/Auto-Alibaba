# 1688 商品自动上传 Skill 设计

## 目标

在个人 Codex Skills 目录创建 `upload-1688-products`，让本机其他 Codex 会话能够通过“快速上传型号 XXX”等请求，稳定调用 `D:\Auto-Alibab` 的现有持久化上传器。

Skill 只负责编排、检查和安全约束，不复制上传器代码，也不修改现有业务实现。

## 安装位置与触发

- 安装目录：`C:\Users\小城\.codex\skills\upload-1688-products`
- 典型触发语：
  - “快速上传型号 W3G710-NU31-03”
  - “上传1688商品 XXX”
  - “继续1688草稿上传”
- 固定项目根目录：`D:\Auto-Alibab`

## 工作流

1. 从用户请求中取得并规范化型号，不猜测相近型号。
2. 确认项目存在，并读取项目 `README.md` 获取当前命令契约。
3. 检查本机 Google Chrome 是否通过 CDP `9223` 运行，以及1688商家工作台是否已登录。
4. 运行 `python -m app.cli doctor --root D:\Auto-Alibab`。
5. 检查Excel精确型号行、PDF、照片和该型号的自动化产物；缺失关键输入时停止并准确报告。
6. 运行 `python -m app.cli run <MODEL> --root D:\Auto-Alibab`。
7. 验证质量错误为零、状态为 `READY_TO_SAVE`，并向用户报告当前页面已停在保存草稿前。

## 固定业务规则

- 型号行不存在时停止。
- 型号行存在但价格或库存为空时，分别使用 `10000` 和 `50`。
- 类目固定为“机械及行业设备 > 风机、排风设备 > 工业风扇 > 其他工业风扇”。
- 发货时效为48小时。
- 包装长宽高取PDF型号图纸，重量取型号重量。
- 运费模板选择“运费”，不选择“8元”。
- 使用本机 Google Chrome、Playwright CDP 和 `9223` 端口。

## 安全边界

- Skill 不点击“保存草稿”、发布或任何等价按钮。
- 只有严格验证 `READY_TO_SAVE`、质量错误为零并确认按钮文本为“保存草稿”后，才报告完成。
- 页面结构变化、登录失效、关键素材缺失或上传失败时停止，不绕过质量检查。
- 不复制或存储1688账号、Cookie、密码等登录凭据。

## Skill 结构

```text
upload-1688-products/
├─ SKILL.md
└─ agents/
   └─ openai.yaml
```

`SKILL.md` 保持简洁，以触发条件、确定性命令、检查顺序和安全边界为核心。上传器自身的详细说明继续以项目 `README.md` 和CLI帮助为准。

## 验证

- RED：记录未安装 Skill 时，其他会话无法自动获得项目路径、命令和安全规则的基线失败。
- GREEN：验证 Skill 元数据、目录结构和内容规范，并模拟典型上传请求确认能定位项目、调用 `doctor`/`run`、停在保存前。
- 回归：运行项目现有测试，确保新增 Skill 不影响上传器。

## 非目标

- 不将项目改造成跨电脑安装包。
- 不把上传器源代码复制进 Skill。
- 不自动创建缺失的产品原始资料。
- 不自动保存草稿或发布商品。
